# Reproducibility bundle

Code and data for *"Geochemistry-informed prediction and layered transferability of
soil organic carbon in the Ganges Delta"* (Uddin et al., submitted to **SOIL**).

Every reported number, table, and figure (except Fig 2, see below) is regenerated by
the scripts here from the data in `data/`. No external drive or network is required
for the core analyses (the cross-delta input is cached in `data/h4_delta_samples.csv`).

## Deposit
- **Code → GitHub** (this repository): `scripts/`, `notebooks/`, `provenance/`, `requirements.txt`, `README.md`, `DATA_SOURCES.md`, and the small CSVs in `data/`.
- **Data → Zenodo** (archived with a DOI, cited in the paper): the full `data/` directory, including the large `data/gis/DEM.tif` (183 MB) which is omitted from GitHub.
- A tagged GitHub release is archived to Zenodo to mint a code DOI.

## Environment
```
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt        # Python 3.11; versions pinned
```

## Reproduce everything
Run from `scripts/` (each writes its outputs there):
```
cd scripts
python3 grothc_model.py            # H2 process model -> grothc_results.csv, grothc_multidelta_ratios.csv
python3 ml_spatial_cv.py           # H3 leak-free spatial CV -> ml_spatial_cv_results.csv
python3 engineered_ablation.py     # H3 engineered-feature ablation
python3 per_zone_correlations.py   # H1 per-zone correlations -> Supplementary Table S10
python3 h4_analysis.py             # H4 transfer matrix + forward/reverse/LODO -> h4_results.json (14 deltas + Ganges; WoSIS + BIS Nederland for Rhine-Meuse)
python3 h4_bis_robustness.py       # H4 robustness: WoSIS-only vs BIS-augmented Rhine-Meuse -> h4_bis_robustness.json
python3 h4_expanded_analysis.py    # H4 expansions: distance-decay, feature universality, few-shot recovery -> h4_expanded_results.json
python3 make_fig3.py               # Fig 3 (climate)        [imports grothc_model]
python3 make_fig4.py               # Fig 4 (ML/DL)
python3 make_fig5.py               # Fig 5 (transferability) [needs h4_results.json + h4_transfer_matrix.csv from h4_analysis.py]
python3 make_figS_h4expanded.py    # SI Fig S1 (distance-decay / universality / few-shot) [needs h4_expanded_analysis.py outputs]
python3 make_fig6.py               # Fig 6 (SI spatial)
python3 create_study_area_figure.py# Fig 1 (study area)     [needs data/gis/, incl. DEM.tif from Zenodo]
```
Run `h4_analysis.py` before `make_fig5.py` (the latter reads `h4_results.json` and `h4_transfer_matrix.csv`), `h4_analysis.py` before `h4_expanded_analysis.py`/`make_figS_h4expanded.py`, and `grothc_model.py` before `make_fig3.py` (which imports it).

The cross-delta panel (`data/h4_delta_samples.csv`) is WoSIS 2023 (Batjes et al., 2024) with the BIS Nederland national survey (4TU.ResearchData) augmenting the Rhine-Meuse cell; both panels (WoSIS-only vs +BIS) are derived from the shipped file's `source` column by `h4_bis_robustness.py`, which confirms the cross-delta transfer failure is not a data-volume artefact.

## Notebooks
Two Jupyter notebooks in `notebooks/` reproduce the analysis with the **H1–H4 code shown inline** — the builder `_build_notebooks.py` embeds each analysis script's source verbatim (no hidden `import`), so the notebooks and `scripts/` cannot drift; figures are rendered by the `make_fig*.py` plotting scripts. Regenerate with `python3 notebooks/_build_notebooks.py`:
- **`01_reproducibility.ipynb`** — clean run-all that regenerates every reported number and figure; intended for peer review and the Zenodo deposit. Shipped with outputs already executed.
- **`02_tutorial.ipynb`** — the same analyses with explanatory narrative and open exploration cells at the end.

Both run top-to-bottom in ≈1 min; Fig 1 is skipped automatically if `data/gis/DEM.tif` is absent.

## Key results these reproduce
| Result | Value | Script |
|--------|-------|--------|
| Baseline mean SOC | 45.1 t C ha⁻¹ | grothc_model.py |
| Management : climate ratio (Ganges) | 5.6× | grothc_model.py |
| Leak-free location-grouped CV R² | 0.69 (vs 0.91 random) | ml_spatial_cv.py |
| Engineered-feature gain ΔR² | +0.05 to +0.11 (GBR) | engineered_ablation.py |
| Cross-delta transfer (210 pairs) | median R² −0.55; 6.2% positive | h4_analysis.py |
| Within-delta CV (median) | +0.25 | h4_analysis.py |
| Forward Ganges→14 deltas | all R² < 0 | h4_analysis.py |
| Reverse pooled(6652)→Ganges | R² = −0.25 | h4_analysis.py |
| Rhine-Meuse within-delta, WoSIS→+BIS | +0.15 → +0.19 (robustness) | h4_bis_robustness.py |
| Distance-decay of transfer | no correlation (|ρ| ≤ 0.11) | h4_expanded_analysis.py |
| Feature-importance universality | mean Kendall τ = +0.38 vs Ganges | h4_expanded_analysis.py |
| Few-shot recovery (all 15 deltas) | 0/15 reach 50% ceiling by n=20 | h4_expanded_analysis.py |

## Notes
- **Fig 2** (geochemistry correlations) has no generator script in this release; it was produced in an earlier analysis stage and is provided as a static image in the manuscript.
- `provenance/wosis_extraction.py` documents how `data/h4_delta_samples.csv` was extracted from the raw WoSIS 2023 snapshot (download required; see `DATA_SOURCES.md`). It is provenance, not needed for the core analyses.
- Random seeds are fixed (`random_state=42`) throughout for exact reproducibility.
- Full data provenance and licences: see `DATA_SOURCES.md`.
